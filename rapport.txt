\chapter{Introduction}
% =============================================================================

\section{Contexte du projet}
CampusEvents AI est une application mobile développée dans le cadre de notre formation en développement mobile. Elle vise à moderniser la gestion des événements universitaires en offrant une plateforme intuitive pour les étudiants et les administrateurs.

L'objectif principal est de centraliser la gestion des événements (conférences, ateliers, compétitions, formations, activités culturelles et sportives) tout en intégrant un assistant IA capable de répondre aux requêtes en langage naturel.

\section{Technologies utilisées}

\subsection*{Backend}
\begin{itemize}
    \item \textbf{FastAPI 0.115.0} --- Framework REST Python moderne et performant
    \item \textbf{SQLAlchemy 2.0.35} --- ORM pour la gestion de la base de données
    \item \textbf{SQLite} --- Base de données légère embarquée
    \item \textbf{Ollama + Llama 3} --- Modèle de langage local pour l'assistant IA
    \item \textbf{Uvicorn 0.30.6} --- Serveur ASGI
    \item \textbf{Pydantic 2.9.2} --- Validation des données
    \item \textbf{HTTPX 0.27.2} --- Client HTTP pour les appels à l'API Ollama
\end{itemize}

\subsection*{Frontend}
\begin{itemize}
    \item \textbf{Flutter 3.44.0 / Dart 3.12.0} --- Framework UI multi-plateforme
    \item \textbf{Riverpod 2.6.1} --- Gestion d'état déclarative
    \item \textbf{GoRouter 14.8.1} --- Routage déclaratif avec redirection d'auth
    \item \textbf{Dio 5.9.2} --- Client HTTP puissant
    \item \textbf{SharedPreferences} --- Stockage local persistant
    \item \textbf{Google Fonts (Poppins)} --- Typographie moderne
    \item \textbf{Material 3} --- Design system Google
\end{itemize}

\subsection*{Infrastructure}
\begin{itemize}
    \item \textbf{Docker / Docker Compose} --- Conteneurisation du frontend web
    \item \textbf{Windows 10+} --- Environnement de développement
    \item \textbf{Android 14} --- Plateforme cible mobile
\end{itemize}

\chapter{Architecture du Projet}

\section{Architecture globale}

Le projet suit une architecture client-serveur classique :

\begin{itemize}
    \item \textbf{Frontend} : Application Flutter (mobile/web) qui communique avec l'API REST.
    \item \textbf{Backend} : API FastAPI qui gère la logique métier, la persistance et l'intégration avec l'IA.
    \item \textbf{IA Locale} : Serveur Ollama avec le modèle Llama 3, hébergé localement.
\end{itemize}

\section{Structure du Backend}

Le backend est organisé selon une architecture en couches :

\begin{verbatim}
backend/
  run.py                        # Point d'entrée (uvicorn)
  app/
    main.py                     # Configuration FastAPI, CORS, routes
    config.py                   # Paramètres (Ollama, DB, credentials)
    database.py                 # Connexion SQLAlchemy
    seed.py                     # Données de démonstration (8 événements)
    models/                     # Modèles SQLAlchemy
      event.py, favorite.py, registration.py, llm_result.py
    schemas/                    # Schémas Pydantic (validation)
      auth.py, event.py, favorite.py, registration.py, llm.py
    repositories/               # Couche d'accès aux données
      event_repository.py, favorite_repository.py,
      registration_repository.py, llm_repository.py
    services/                   # Logique métier
      auth_service.py, event_service.py, favorite_service.py,
      registration_service.py, llm_service.py
    routers/                    # Endpoints API
      auth.py, events.py, favorites.py, registrations.py, llm.py
\end{verbatim}

\chapter{Intégration de l'Intelligence Artificielle}
% =============================================================================

\section{Architecture de l'assistant IA}

Le flux de traitement d'une requête IA suit les étapes suivantes :

\begin{enumerate}
    \item L'utilisateur envoie une requête en langage naturel depuis l'application Flutter.
    \item Le backend FastAPI reçoit la requête via POST /api/llm/query.
    \item Le \texttt{LlmService} construit un contexte à partir des événements disponibles (jusqu'à 50) ainsi que des favoris et inscriptions de l'utilisateur.
    \item Un prompt spécialisé est généré selon le type de requête (search, recommendation, planning, qa).
    \item Le prompt est envoyé à Ollama (http://localhost:11434/api/generate) avec le modèle Llama 3.
    \item La réponse est analysée, sauvegardée dans la table \texttt{llm\_results} et renvoyée à l'utilisateur.
\end{enumerate}

\begin{figure}[H]
\centering
% PLACEHOLDER_SCREENSHOT: Schema_Flux_IA
% Insérez le diagramme de flux de l'assistant IA
% \includegraphics[width=\textwidth]{screenshots/ia_flow.png}
\caption{Flux de traitement d'une requête IA}
\end{figure}

\section{Exemple de prompt --- Mode Recherche}

\begin{lstlisting}[frame=single, language=Python, basicstyle=\small\ttfamily]
prompt = f'''
Tu es un assistant IA specialise dans les evenements
universitaires du campus.

Contexte des evenements disponibles:
{contexte}

Question de l'etudiant: {requete}

Cherche dans le catalogue les evenements qui correspondent
a cette requete. Reponds en francais de maniere claire
et structuree en listant les evenements pertinents avec
leurs details (titre, date, lieu).
Si aucun evenement ne correspond, propose des alternatives.
'''
\end{lstlisting}

\chapter{Installation et Déploiement}
% =============================================================================

\section{Prérequis}

\begin{itemize}
    \item \textbf{Python 3.10+} --- Pour le backend FastAPI
    \item \textbf{Flutter 3.16+} --- Pour le frontend mobile
    \item \textbf{Ollama} --- Serveur LLM local avec le modèle \texttt{llama3}
    \item \textbf{Android SDK} (API 34+) --- Pour le déploiement mobile
    \item \textbf{Docker Desktop} (optionnel) --- Pour le frontend web conteneurisé
\end{itemize}

\section{Lancement du projet}

\subsection*{1. Backend FastAPI}
\begin{lstlisting}[frame=single, basicstyle=\small\ttfamily]
cd backend
python -m venv venv
venv\Scripts\activate    # Windows
pip install -r requirements.txt
python run.py
# API disponible sur http://localhost:8000
\end{lstlisting}

\subsection*{2. Serveur Ollama}
\begin{lstlisting}[frame=single, basicstyle=\small\ttfamily]
ollama run llama3
# API disponible sur http://localhost:11434
\end{lstlisting}

\subsection*{3. Frontend Flutter}
\begin{lstlisting}[frame=single, basicstyle=\small\ttfamily]
cd frontend
flutter pub get
flutter run                # Lancement interactif
# ou sur un appareil specifique :
flutter run -d R5CW60P81PJ # Exemple avec ID de telephone
\end{lstlisting}

\subsection*{4. Frontend Web avec Docker}
\begin{lstlisting}[frame=single, basicstyle=\small\ttfamily]
cd frontend
docker compose up
# Application disponible sur http://localhost:3000
\end{lstlisting}
